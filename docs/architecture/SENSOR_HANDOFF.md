## Contracts honored (from INFERENCE_SERVICE_HANDOFF.md)

- Emits the exact 56-feature dict, exact names/order/units, RAW values (no scaling;
  the service scales only for the AE internally). FEATURE_ORDER is hardcoded in
  `flow_features.py` and asserted by a test.
- Fidelity target is the **Engelen/Distrinet "improved" CICFlowMeter**, NOT stock —
  the dhoogla/cicids2017 training parquet was generated by the Engelen fork. Key
  source-verified definitions reproduced: packet length = L4 payload bytes; all
  time fields in microseconds; sample (n-1) std/variance; Down/Up = true float
  division; TCP flow ends on mutual FIN or RST (else 120s timeout), UDP on timeout;
  Active/Idle populated ONLY by intra-flow gaps > 5s (endActiveIdleTime is dead code
  in the fork) so most flows have Active=Idle=0; flag-count features emitted BINARY
  0/1 (dhoogla cleaning binarized them). INIT_WIN_DEFAULT = -1 (confirmed from data).
- POSTs `{"features": {...56...}}` to `POST /predict`. Consumes the documented response.

## Live findings (these are RESULTS, record them)

### 1. Autoencoder domain-shift false-positive flood (the headline)
On local (home/office WiFi) benign traffic, the AE at its shipped threshold 0.0726
(95th pct of CICIDS2017 benign reconstruction error) flags **75.2% of benign flows
as anomalous** (152/202; confirmed ~78% on the larger live run — two captures agree).
Local score distribution: median 1.12, p90 28.6, p95 30.4, p99 39.5, max 47.9. A
local p95 threshold (~30.4) is ~419x the trained value. Interpretation: this is the
live, quantified extension of the project's cross-dataset transfer-failure thesis —
the AE's learned "normal" (CICIDS2017) is mismatched enough to this network that
reconstruction error stops cleanly separating normal-here from attack. NOT a bug.

### 2. Supervised models stay silent live
Across all live runs, supervised consensus produced ~0 attack alerts on benign
(one spurious Infiltration in ~1500 flows). Opposite symptom, same root cause as the
AE: out-of-distribution -> classifiers default toward benign (risk = false negatives).

### 3. Live attack test — pipeline passed, no supervised alert (with caveat)
nmap connect-scans against gateway/LAN host while sensor live: pipeline carried the
burst (12,676 pkts, dropped=0, errors=0), but attacks=0. CAVEAT: targets were almost
entirely *filtered* (no response), so most scan probes are lone outbound SYNs ->
single-packet flows dropped at aggregation (MIN_PACKETS_TO_EMIT=2) and never reach
the classifier. Honest claim: "pipeline classified live scan traffic end-to-end with
zero loss; no supervised PortScan alert fired." NOT: "models definitively miss live
PortScan" — the stimulus was weak (filtered host + min-packet threshold). This
filtered-host/min-packet blindness is itself a legitimate discussion point.

## Locked decisions (don't relitigate)

- **Alerting split (the design that makes the original system usable without retraining):**
  supervised `is_attack` -> Telegram pager stream; AE `is_anomalous` -> Redis/dashboard
  research stream. AE threshold STAYS at 0.0726 for the original system; report the
  75.2% as the finding. Do NOT page on the AE (would flood).
- AE retrain on local benign is the *fix*, deliberately DEFERRED to after the original
  system is finalized (separate effort; unsupervised, Kaggle, reuses capture pipeline).
- Supervised retrain on local attacks is explicitly OFF the table (needs labeled local
  attacks — infeasible/unwise to generate; tiny sample). Supervised models stay as the
  comparison baseline that demonstrates the transfer finding.
- Redis Streams over Kafka. venv = local dev truth; Docker image = deploy truth.
- Capture vantage = AF_PACKET raw socket (matches tcpdump validation reference).

   ALERT/ANOMALY lines but the alert payload carries NO source/dest identity, so
